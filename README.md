## 阿里云：国内应用首选

![阿里云ECS服务器](image/63762807135902.webp)

**ECS云服务器配置要点：**
- 不建议勾选"免费安全加固"，避免额外的安全组件负载
- 快速部署可优先使用"自定义密码"方式

- IPv6配置后默认仅内网访问，公网访问需购买"IPv6公网带宽"

**VPC网络架构：**
每个交换机管理单个可用区，跨区部署需新增对应可用区的交换机，且网段范围不能重叠。例如可用区K使用192.168.0.0/24，其他可用区需使用192.168.1.0/24。

**RDS数据库实例：**
- 默认禁止外部访问，需添加白名单或直接"全部开放"
- 支持CNAME至实例主机名，通过自定义域名连接
- 默认已启用lower_case_table_names=1
- 修改character_set_server=utf8mb4支持Emoji（会自动重启）

**实测优势：**
联通宽带上传100MiB文件至阿里云ECS仅需2分30秒，远优于美国机房的5-6分钟，适合需要频繁与国内用户交互的应用。

## 腾讯云：注意隐藏费用

**COS对象存储的坑：**
查看外网流量访问记录（IP等信息）必须开通付费的日志服务（CLS）。COS本身开启日志管理不收费，但CLS服务的日志存储和检索分析会产生费用。即使关闭日志检索功能，历史日志存储仍会持续计费直至手动删除。

**建议操作：**
在COS控制台关闭日志检索后，务必前往CLS控制台手动删除对应的日志主题（名称为cos-log-store）。或者设置日志存储至指定存储桶，避免开通付费的"日志检索"服务。

## 自建CDN方案



如果你有多台服务器且流量较大，可以考虑自建CDN降低成本。

**Nginx配置优化：**
- 缓存命中头：X-Cache-Status: HIT 或 CF-Cache-Status: MISS
- 配置`proxy_cache_lock on`使除首次访问外的请求全部挂起
- 大文件优化：设置`slice 1m`支持HTTP Range请求，适配播放器按需下载
- 可选监控：NGINX Amplify Agent可视化

**专用CDN服务器：**
可考虑Apache Traffic Server等专业CDN软件。
